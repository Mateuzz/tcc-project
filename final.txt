    pos processing -> babylon melhor ou talvez three 

funcoes especificas (cientifico, lego, CAD) -> three
    caso contrario babylon. three  vai servir
    
praticidade de uso -> playcanvas
    geracao de lightmaps

para jogos(iniciantes)
    playcanvas (pela performance, praticidade de uso, self hosting) exceto se pos processamento
        babylon
    playcanvas -> babylon.js
    para jogos super realistas com path tracing -> babylonjs 

para aplicacoes complexas de alta performance em geral
    playcanvas -> segundo lugar babylon

para aplicacoes extras, como pequenos efeitos em sites e cenas simplas
    threejs pelo peso

para cads ou aplicacoes cientificas
    threejs (com menos funcionalidades de baboseira e ser mais leve)
    melhor suporte a usuarios com menor conexao e pcs piores

para pcs mais fracos
    threejs

para evitar bug de sombra evite a threejs

para infraestrutura limitada ou usuarios com conexao lenta
    threejs, seguidoda playcanvas
